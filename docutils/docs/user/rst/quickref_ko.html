Inline Markup

(details)

Inline markup 텍스트 내의 단어(word)와 구(phrases)가 문자 스타일(이탤릭체와 볼드체)과 하이퍼링크 같은 기능을 가질 수 있게 해준다.

Plain text Typical result Notes
*emphasis* emphasis 이탤릭체로 표시된다.
**strong emphasis** strong emphasis 볼드체로 표시된다.
`interpreted text` (오른쪽의 노트 참조) Interpreted text의 렌더링과 의미는 도메인 또는 애플리케이션에 종속되고, 인덱스 항목이나 프로그램 식별자 같은 명시적으로 설명된 마크업에 사용할 수 있다.
``inline literal`` inline literal 일반적으로 단일 공백의 텍스트로 렌더링된다. 공백은 유지되어야 하지만, 줄바꿈은 유지되지 않을 것이다.
reference_ reference 하나의 단어를 이용한 하이퍼링크. Hyperlink Targets 참조.
`phrase reference`_ phrase reference 공백 또는 구두점을 이용한 하이퍼링크는 따옴표를 묶어야한다. Hyperlink Targets 참조.
anonymous__ anonymous 두 개의 밑줄을 이용해, 단순하거나 구로 이루어진 reference는 anonymous일 수도 있다. (reference 텍스트는 대상에서 반복되지 않는다.). Hyperlink Targets 참조.
_`inline internal target` inline internal target 텍스트 내의 상호 참조 대상. Hyperlink Targets 참조.
|substitution reference| (오른쪽의 노트 참조) substitution definition 으로부터 바뀐 결과. 이는 텍스트, 이미지, 하이퍼링크 또는 다른 대상들의 결합일 수도 있다.
footnote reference [1]_ footnote reference 1 Footnotes 참조.
citation reference [CIT2002]_ citation reference [CIT2002] Citations 참조.
http://docutils.sf.net/ http://docutils.sf.net/ 독립 실행형 하이퍼링크.

Asterisk(*), backquote(`), vertical bar(|) 와 underscore(_) 는 inline delimiter 문자이다. Asterisk, backquote, and vertical bar 는 따옴표처럼 사용된다; 짝을 이룬 문자들은 marked-up 단어나 구를 둘러싸고, 그 밖에는 공백이나 다른 따옴표(quoting)가 필요하다. 그리고 짝을 이룬 문자들로 둘러 쌓여진 단어나 구 안에서 공백이 있으면 안된다. inline 구분 문자를 문자 그대로 사용하려는 경우, escape (\) or quote(``)를 사용한다.

상세하게, the reStructuredText 설명서에는 inline markup에서 다음의 규칙들이 start-strings과 end-strings (inline markup delimiters)에 적용된다고 나와있다. :

  1. Start-string은 텍스트 블록을 시작하거나 공백이나,   ' " ( [ { or < 보다 앞에 있어야 한다.
  2. Start-string 바로 뒤에 공백이 없어야 한다.
  3. End-string 바로 뒤에 공백이 없어야 한다.
  4. End-string은 텍스트 블록(문서 끝 또는 빈 줄의 끝)을 끝내거나 공백이나,  ' " . , : ; ! ? - ) ] } / \ or > 보다 앞에 있어야 한다.
  5. Start-string이   ' " ( [ { or < 중 하나보다 바로 앞에 온다면, 대응하는 문자 ' " ) ] } or > 보다 뒤에 올 수 없다.
  6. End-String은 Start-string으로부터 적어도 하나의 문자에 의해 구분되어야 한다.
  7. Start-string 또는 end-string보다 앞에 있는 unescaped backslash 는 inline literal의 end-string을 제외하고는 markup을 인지할 수 없다.

inline markup이 중첩되지 않을 수도 있다는 점을 기억한다. (inline literal이 다른 inline markup delimiter 문자를 포함할 수 있다는 점을 제외하고)

Escaping with Backslashes

(details)

reStructuredText는 markup 문자에 주어진 특별한 의미를 오버라이드 하고 literal 문자를 그대로 쓰기 위해 backslashes ("\")를 사용한다. literal backslash를 쓰기 위해, escaped backslash를 사용하면 된다. ("\\"). 예:

Raw reStructuredText Typical result
*escape* ``with`` "\" escape with ""
\*escape* \``with`` "\\" *escape* ``with`` "\"

파이썬 문자열에서는 backslash 문자들을 실제로 보이기 위해서 escape를 하는 것이 필요할 것이고, 가장 간단한 방법은 raw string을 사용하는 것이다.

Python string Typical result
r"""\*escape* \`with` "\\"""" *escape* `with` "\"
 """\\*escape* \\`with` "\\\\"""" *escape* `with` "\"
 """\*escape* \`with` "\\"""" escape with ""

Explicit Markup

Explicit markup block은 hyperlink, targets, comments와 같은 직접적으로 문서를 참조하지 않거나 directives같은 특별한 처리를 요구하는 footnote를 구성하는 데에 사용된다. 모든 Explicit markup block은 2개의 period와 공백문자로 시작해야한다.

'autonumber labels'를 부여받을 수도 있다 - 예를 들면 #fourth]_ 와 [#third]_.

'autonumber labels'를 부여받을 수도 있다. - 예를 들면, 4 와 3.

[1] This is the first one.
[2] This is the second one.
[3] a.k.a. third
[4] a.k.a. fourth
이렇게 사용하면 [*]_ 과 [*]_ 처럼 자동 symbol footnote도 가능하다.

.. [*] This is the first one.
.. [*] This is the second one.

이렇게 사용하면 * 과 †처럼 자동 symbol footnote도 가능하다.

[*] This is the first symbol footnote
[†] This is the second one.

자동으로 넘버링된 footnote의 넘버링은 reference가 아닌 footnote의 순서에 의해 결정된다. autonumber labels ("[#]_")없이, 자동으로 넘버링된 footnote references들에 대해, references와 footnotes 같은 상대적 순서를 가지고 있어야 한다. auto-symbol footnotes ("[*]_")에 대해서도 유사하다.

Hyperlink Targets

(상세 내용)

External Hyperlink Targets

Plain text Typical result
External hyperlinks, like Python_.

.. _Python: http://www.python.org/

Fold-in form
External hyperlinks, like Python.
Call-out form
External hyperlinks, like Python.

Python: http://www.python.org/

"Fold-in"은 일반적으로 HTML 문서에서 사용되는 표현방법이다. (케이크에 들어가는 재료처럼 indirect hyperlink를 "folded in"으로 생각해라.) 그리고"call-out"는 링크가 명시적으로 표현되어질 필요가 있는 printed 문서에 더욱 적합하다. footnote가 하나의 예이다. "target-notes" directive을 사용해서 call-out 형태를 강제로 사용하게 할 수도 있다.

reStructuredText는 가독성의 측면에서 편리함을 위해 embedded URIs (상세 내용)을 제공한다. Hyperlink reference는 직접 angle brackets 내의 target URI를 inline으로 포함할 수 있다. 다음의 위와 같은 예이다.:

Plain text Typical result
External hyperlinks, like `Python
<http://www.python.org/>`_. 		External hyperlinks, like Python.

Internal crossreferences, like example

이는 crossreference target의 예이다.

Indirect Hyperlink Targets

(상세 내용)

Plain text Typical result
Python_ is `my favourite
programming language`__.

.. _Python: http://www.python.org/

__ Python_

Python is my favourite programming language.

"__"로 시작하는 라인인 두번째 hyperlink target은 an indirect hyperlink target (간접적으로 "Python_" reference를 통해 Python 웹사이트를 가리킴)이자 anonymous hyperlink target 이다. . 텍스트에서, a double-underscore 접미사는 anonymous hyperlink reference를 표시하는데 사용된다. an anonymous hyperlink target에서 reference 텍스트는 반복되지 않는데, 이는 긴 텍스트를 가진 reference나 일회용 reference에 유용하게 사용된다. 하지만 target은 reference가 싱크에서 벗어나는 것을 막기위해 가까이에서 유지되어야 한다.
